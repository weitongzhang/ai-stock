# backtesting

Adapters and schemas for connecting external backtest engines to the trading
research system.

Backtest execution should be deterministic. Agent-based judgment belongs in the
evaluation layer after backtest results have been produced.

## Contract

- `BacktestConfig`: strategy name, date range, universe, cash, fees and params.
- `SignalEvent`: normalized buy/sell/wait events generated by stock-analysis
  domains such as YC-buy or BSA/LIFT.
- `BacktestEngine`: protocol implemented by any concrete backtest engine.
- `BacktestResult`: normalized performance report, trades, equity curve and
  source signals.

## Extension point

Wrap the existing user backtest code with `BacktestEngine.run(...)`. Keep the
old engine responsible for deterministic execution, then use the Harness eval
layer to review whether the strategy result is acceptable.
