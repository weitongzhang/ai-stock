# 知识检索与回答路由

## 首选方式

使用本地检索器先召回相关条目：

```bash
python3 scripts/search_knowledge.py "追高怎么判断" --limit 8
python3 scripts/search_knowledge.py "商业航天 可回收火箭" --limit 8
python3 scripts/search_knowledge.py --id MX-0135
python3 scripts/search_knowledge.py "粉丝收益" --include-excluded
```

检索结果包含条目编号、日期、主题、归因置信度和原文摘要。读取结果后，再按条目编号回到完整目录核验语境。

默认检索会隐藏人工隔离条目，并在相关性相近时优先返回归因置信度较高的语录。只有审计材料覆盖、研究归因风险时，才使用 `--include-excluded`。

部分集大成长条目内部跨越多个日期。检索器会在能够识别时同时显示“片段日期”；历史观点与预测核验应优先使用片段附近日期，并回到完整原文确认。

## 问题路由

| 用户问题 | 优先检索 | 必须补充 |
|---|---|---|
| 核心思想、完整体系 | `core-viewpoints`、`argumentation-templates` | 核心原文条目 |
| 买点、卖点、追高、被套 | 直接检索问题关键词 | `extended-methodology-map` |
| 行业、板块、个股历史观点 | 行业名、公司名 | 日期、归因置信度、最新事实 |
| 历史预测准确性 | `prediction-ledger` 与具体关键词 | 完整原文、事后数据 |
| 观点矛盾或变化 | 同一主题分日期检索 | `contradictions-and-evolution` |
| 表达风格或类比 | `analogy-and-expression-library` | 对应完整原文 |

## 证据优先级

1. 语录合集中的完整转录：较高归因置信度。
2. 集大成材料中明确标注的本人回复：中高归因置信度。
3. 集大成材料中未明确归因的转录或整理：中低归因置信度。
4. 网友观点、收益转述、个股荐股：保存但不得归因。

## 最小证据要求

- 方法论回答：至少召回2条不同日期或不同场景的支持材料。
- 具体行业回答：至少召回1条原始观点，并核验最新事实。
- 声称观点变化：至少列出变化前后两条带日期材料。
- 声称预测准确或错误：必须给出预测日期、原话、验证日期和客观数据。
